Configuration

Scope

The goal of this chapter is to start instrumentig our components based on Config Admin. At this time, only the method of creating, updating and storing configuration will be introduced.

The business logic and the integration tests were developed in the last chapter. If you skipped that part, please check out the 02 - tdd branch in eclipse and import the content as a maven project.

Step 0: Adding Jetty and Webconsole

We would like to allow the programmers to configure the UserServiceComponent. The component will be instantiated only when there is a configuration available. Also, we would like to make it possible to instantiate the component more than once by creating multiple configuration entries. Multiple instances will be useful when tests should run on different database engines.

Open the UserServiceComponent class and change the annotation of the class in the following way: 

@Component(configurationFactory = true)
@Properties({ @Property(name = "service.description", propertyPrivate = false) })
@Service
